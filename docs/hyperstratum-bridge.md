# Hyperstratum Bridge

Force of Will is downstream of Hyperstratum.

This repository defines a six-component model of structured agency:

```txt
W_a(t) = <P_a(t), N_a(t), F_a(t), I_a(t), V_a(t), R_a(t)>
```

Hyperstratum defines the upstream recursive-form ontology that supplies terms such as `hypernode`, `person-hypernode`, `recursive-graph`, `self-graph`, `mira-graph`, `self-mira-recursive-graph`, and `social-mira-graph`.

Repository:

```txt
https://github.com/TimeLordRaps/hyperstratum
```

## Local import rule

In this repository, `hypernode` should be read only as an imported upstream term:

```txt
hypernode = a local recursive agency-node that can be modeled through the six-will structure.
```

The full ontology belongs to Hyperstratum, not Force of Will.

## Person-hypernode bridge

A person-hypernode may be modeled through Force of Will when the person is treated as a will-bearing structured agency-object:

```txt
PersonHypernode -> W = <P, N, F, I, V, R>
```

Force of Will does not define personhood by social reception.

Hyperstratum's identity distinction should be preserved:

```txt
self-mira-recursive-graph != social-mira-graph
```

Meaning:

```txt
A person's self-representation is not the same as society's reception, labeling, interpretation, or misrepresentation of that person.
```

## Force of Will import boundary

Force of Will may reference:

```txt
hypernode
person-hypernode
self-mira-recursive-graph
social-mira-graph
hyperrelation
hyperemergent
hyperrepresentation
```

But it should not redefine:

```txt
hypertopology
hypermanifold
hyperfield
hyperstructure
hyperform
full hyperrepresentation ontology
```

Those belong upstream in Hyperstratum.

## Practical rule

When a term from Hyperstratum appears in Force of Will, use it only as much as needed to clarify agency modeling.

Do not let the Force of Will repo become a second, divergent hyperstratum.
